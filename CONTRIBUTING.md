# Contributing to The Stella Protocol

Thank you for considering contributing to the Stella Protocol.

## How to Contribute

### Reporting Issues
- Use GitHub Issues for bug reports and feature requests
- Include steps to reproduce for bugs
- For feature requests, explain the use case

### Pull Requests
1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes
4. Ensure all existing functionality still works
5. Submit a PR with a clear description of the change

### What We're Looking For
- Bug fixes in the CLI installer
- Improvements to satellite definitions
- New Punk Records templates
- Documentation improvements
- Compatibility with additional AI tools

### Guidelines
- Keep PRs focused on a single change
- Follow existing code style
- Update documentation if behavior changes
- Test the CLI installer before submitting

## Community Satellites (Grand Fleet)

We welcome community-contributed satellites. If you have a specialized role that would benefit the protocol:

1. Create a satellite definition following the format in `protocol/satellites/`
2. Explain when it activates and what it does
3. Submit as a PR with a clear use case
