# brain/ — Project Memory

Files in this directory are Stella Protocol's persistent state. The CLAUDE.md session hook reads `log-pose.md`, `scope-changes.md`, and `vivre-cards.md` at every session start.

## Active files

| File | Purpose | Who writes |
|------|---------|-----------|
| `log-pose.md` | Current phase, heading, active work, buster call status | `punk-records` |
| `project-context.md` | Tech stack, critical constraints, non-negotiables, key file locations | Stella (filled during DEFINE) |
| `vivre-cards.md` | Decision log (v0.8.0+). Append-only. | `punk-records`, `buster-call` |
| `scope-changes.md` | Cipher Pol scope drift log | `cipher-pol` |
| `architecture.md` | Stack decisions, data model, API surface | `pythagoras-research`, `punk-records` |
| `test-plan.md` | Manual QA checklist | `stella-review` (Lilith Blue) |
| `ideas.md` | Parked / approved / killed ideas | `shaka-brief` (IMU) |
| `taskplan-[name].md` | Bite-sized task checklist for a PRD — generated by `atlas-taskplan`, consumed by `stella-parallel` | `atlas-taskplan` |
| `observability-plan.md` | Monitoring plan (Status: Planned) | `stella-close` (York) |

## Archive

| File / Dir | Contents |
|-----------|---------|
| `vivre-cards-archive.md` | Decision log v0.4.2–v0.7.1 |
| `archive/` | Shipped PRDs, completed artifacts, triaged documents |

## Lifecycle rules

- Files in `archive/` are never edited — historical record only
- `vivre-cards.md` is append-only — past entries are never modified, only new entries appended
- When `vivre-cards.md` exceeds ~150 lines, invoke `punk-records` to create a version boundary and archive older entries to `vivre-cards-archive.md`
